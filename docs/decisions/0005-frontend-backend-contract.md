# 0005. The frontend/backend contract: generated types, one IPC module, a scripted mock

## Status
Accepted (2026-07-23)

## Context

PROJECT.md §3.2 names three seams: `types.gen.ts` generated from Rust,
`backend.ts` as the only `invoke()`/`listen()` caller, and a MockBackend the UI
and the E2E suite develop against (issue #4). Making them real forced four
decisions.

1. **How generation runs.** tauri-specta can export bindings at app startup in
   dev, or on demand. Startup export means the file only refreshes when someone
   runs the app.
2. **How capabilities are typed.** A Rust enum of `CAP_*` values would be
   type-safe, but a newer `headsetcontrol` reporting an unknown capability would
   then fail to deserialise — the device would vanish instead of losing one row.
3. **How the "only backend.ts talks to Tauri" rule is enforced.** The issue
   suggests a lint rule or a CI grep.
4. **How the mock is selected**, for `make dev` and for Playwright (#13).

## Decision

- **Bindings are generated by a Rust test** (`export_typescript_bindings` in
  `lib.rs`, run by `make gen`). Because it is a test, `make rs-check` regenerates
  on every CI run, and `make gen-check` fails the build when the checked-in
  `types.gen.ts` differs — a stale contract cannot merge. The IPC surface is
  described exactly once, in `lib.rs::ipc()`, which both registers the commands
  with Tauri and generates their TypeScript.
- **Capabilities stay `Vec<String>` / `string[]`** end to end. Rust does not
  interpret them (PROJECT.md §3.1), and an unknown capability has to be
  ignorable, not fatal. Parameters are addressed by the same identifiers, so
  `ParamValue` (`Int | Bool`) is all the write path needs.
- **The rule is an ESLint rule**, not a grep: `no-restricted-imports` blocks
  `@tauri-apps/api*` everywhere under `src/` except `src/core/backend.ts`. It
  fails in the editor, not only in CI, and it cannot be defeated by formatting.
- **`VITE_BACKEND=mock` selects the MockBackend** (`make dev-mock`), and the
  instance is parked on `window.__headsetDeckMock` so Playwright can script
  hotplug, latency and failures from the page. Failures are modelled as either a
  typed error or a call that never settles — the hung-backend case the UI must
  survive.
- **`commands.rs` is excluded from the Rust coverage gate.** Each command is a
  one-line delegation to the backend behind Tauri's managed state; unit-testing
  it would test Tauri, and the real IPC path is covered by the smoke E2E (#14).

## Consequences

- Changing a Rust type is a one-command ritual (`make gen`) and impossible to
  forget: CI regenerates and diffs.
- The UI never learns whether it is talking to Rust or to the mock, which is what
  makes deterministic E2E possible without hardware.
- Unknown capabilities cost type safety in TS: the feature registry (#12) keys on
  strings and must handle a miss by ignoring the row.
- `UnimplementedBackend` is registered until the adapter lands (#8), so the real
  IPC path answers `not_implemented` rather than pretending to work.
- The mock lives in the production bundle. It is a few kilobytes and the flag is
  build-time; if that ever matters, the import becomes dynamic.
